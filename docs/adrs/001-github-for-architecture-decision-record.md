# ADR 001: GitHub for architecture decision record

## 📝 Context and Problem Statement

We would like to hold documentation, architecture decisions, ... on a public platform.

Putting technical documentation in GitHub has additional benefits:

- Using pull requests gives us a clear review and approval process
- It is part of the same workflow as other activities that we do on a day to day basis (e.g. writing code)
- The information can be held in the open and viewed by anyone

## 💡 Decision Outcome

1. Our architecture decisions log for the Portefaix platform will be held in GitHub

## 📈 Consequences

Our architecture decisions will be created into this repository: [portefaix-adr](https://github.com/portefaix/portefaix-adr)
