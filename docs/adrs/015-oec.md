# ADR 015 - New observability stack: OEC

## 📝 Context and Problem Statement

The three new elements in the observability echosystem:

- [OpenTelemetry](https://opentelemetry.io/)
- [eBPF](https://ebpf.io/)
- [ClickHouse](https://clickhouse.com/)

Stack integration:

- Data collection: OpenTelemetry collects application traces, metrics and logs.
- System monitoring: eBPF monitors system performance and kernel-level events.
- Storage and analysis: data collected by OpenTelemetry and eBPF is sent to ClickHouse for storage and analysis.

This observability stack enables comprehensive and detailed monitoring of application and system performance, providing DevOps and SRE teams with valuable insights for debugging, optimization and maintenance.

## Considered Options

1. [GroundCover](https://www.groundcover.com/)
2. [Signoz](https://signoz.io/)
3. [OneUptime](https://oneuptime.com/)
4. [Qryn](https://qryn.metrico.in/)
5. [HyperDX](https://www.hyperdx.io/)

## Pros and Cons of the Options

### GroundCover

- 🚫 Not Open Source
- ✅

### Signoz

- ✅ Open Source
- ✅ Lot of [dashboards](https://github.com/SigNoz/dashboards)
- 🚫 No EBPF support
- 🚫 arm64 not supported

### OneUptime

- ✅ Open Source
- 🚫 No EBPF support

### Qryn

- ✅ Open Source
- ✅ Compatible with Loki, Prometheus, Tempo, Pyroscope
- 🚫 Not a large community

### HyperDX

- ✅ Open Source
- 🚫 No EBPF support

## 💡 Decision

Cancelled.
